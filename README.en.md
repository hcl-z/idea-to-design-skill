# Idea to UI Design Skill

[中文版](./README.md)

A skill that turns a user's rough idea into a frontend UI and interaction design document.

## Workflow

This skill is primarily based on the superpower `brainstorming` skill and keeps its core workflow:

- Understand the context first
- Ask one key question at a time
- Propose 2-3 directions and compare trade-offs
- Present the design in sections and get approval
- Produce a design document
- Hand off to planning or implementation only when needed

## Current Focus

This skill focuses on `idea -> frontend UI / interaction design`, with outputs centered on:

- A complete page inventory
- Layout and module structure for each page
- User flows
- Key interactions and state design
- Visual direction and a compact design system summary

## What It Does Not Cover

This skill does not produce:

- Backend architecture
- API design
- Database or data model design
- Concrete code logic
- Engineering implementation plans

## Output Format

The final output is a Markdown design document for frontend UI and interaction design, not a technical implementation document.
